Lookup decomposition

This gadget makes use of a K-bit lookup table to decompose a field element α into K-bit words. Firstly, express α=k0​+2K⋅k1​+22K⋅k2​+⋯, where each ki​ a K-bit value.

We initialize the running sum z0​=s, and compute subsequent terms zi​=2Kzi−1​−ki−1​​. This gives us:

z0​z1​z2​​=α=k0​+2K⋅k1​+22K⋅k2​+23K⋅k3​+⋯,=(z0​−k0​)/2K=k1​+2K⋅k2​+22K⋅k3​+⋯,=(z1​−k1​)/2K=k2​+2K⋅k3​+⋯,⋮​

Each K-bit word ki​=zi​−2K⋅zi+1​ is range-constrained by a lookup in the K-bit table. This gadget takes in α,n and returns (z0​,z1​,⋯,zn​).

Strict mode

Strict mode constrains the running sum output zn​ to be zero, thus range-constraining the field element to be within n⋅K bits.

Short range check

Using two K-bit lookups, we can range-constrain a field element α to be n bits, where n≤K. To do this:

  1. Constrain 0≤α<2K to be within K bits using a K-bit lookup.
  2. Constrain 0≤α⋅2K−n<2K to be within K bits using a K-bit lookup.
